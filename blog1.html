Quantum Computing

Unlike classical computers that use binary characters, 0 and 1, also called bits, quantum computers use something called qubits. Bits in classical computers can have only two states, either 0 or 1, whereas qubits can have more than two states, called as a superposition state.

Superposition

Superposition state can be thought of as a probability distribution having more than one state. Once it is measured at any given point, it either points towards 0(up spin) or 1(down spin), and the measured output may vary each time. Suppose, there are n states for a certain qubit, then the sum of probabilities of all those states will always be equal to 1.

In quantum computing, we use Bloch sphere to geometrically represent the superposition state of the qubit. It is also called a pure state space of a 1 qubit quantum register. A superposition state can lie on any point on this sphere, but when the state is measured, it either shows 0 or 1. Since Qubits can be in more than one state until it is measured, the state cannot be represented as 0 or 1 as we do for classical bits. So, we use two orthogonal vectors to represent qubits, and Bra-ket notation is used.

Dirac Notation

Here, it shows the matrix representation of the dirac notations|0⟩ and |1⟩, and these are used to represent up spin and down spin respectively, spin means a linear combination of up spin and down spin, and coefficient alpha is the amplitude.

In classical computers, we use gates to perform operations on bits. Similarly, we have a different set of gates to perform operations on qubits, and unlike classical gates, these are reversible.
Example: Hadamard, Pauli , Controlled, etc.

Entanglement

It is a property of quantum particles where when two entities are entangled, we can say that they are no longer independent, and any changes in one of the entities are reflected in the state of the other entity. This phenomenon is not normal in the classical world.

Qiskit and IBM Quantum Systems

The workflow of Qiskit involves three steps:

  1. Build - design a quantum circuit.
  2. Execute - run the circuits on various backends, which includes simulators as well as systems.
  3. Analyze - calculate summary statistics and visualize the experiments.
Qiskit has four elements:
  1. Terra: It is the foundation on which the rest of the Qiskit runs. It works at the level of circuits and pulses.
  2. Aer: It consists of simulators, emulators, and debuggers, which helps by mimicking quantum computation. It has C++ simulator backends for executing circuits compiled in Terra, and it can provide realistic noise simulations that occur on real quantum devices.
  3. Ignis: It is used to design error correction codes. It includes better characterization of errors, improving gates, and computing in the presence of noise.
  4. Aqua: This is used to build algorithms for quantum computers, and solve real-world problems.
In Qiskit programming, the first thing is to learn is to build QuantumCircuit. We should begin by defining qubit registers and perform the calculation on those using quantum gates. To store the output we define classical bits, then using Qiskit visualization tools we can represent the circuit and the output graphically.
